# 出行决策Agent - 项目想法说明

## 1. 项目背景与痛点

### 用户场景
- 计划周六出去玩
- 发现上海在下雨
- 想知道周边哪些城市不下雨
- 还要找这些城市有什么好玩的、好吃的

### 现有痛点
- 没有工具能概览周边城市的天气情况
- 出行方案需要分别查天气、查路线、查攻略
- 信息分散，决策成本高

## 2. 产品愿景

打造一个**智能出行决策Agent网页**，帮助用户快速决定去哪里玩。

## 3. 产品路线图

### 阶段一：轻度版 - 天气智能决策（MVP）
**目标**：做一个真正能用的产品，解决"周末去哪里不下雨"的核心痛点

**成功标准**：
- ✅ 功能完整：输入→计算→展示→交互 全流程可用
- ✅ 性能优秀：5秒内给出结果
- ✅ 体验良好：界面美观、交互自然、无卡顿
- ✅ 稳定可靠：不是Demo，是真实可用的工具

**核心功能**：

**1. 智能输入**
- Agent对话式交互：
  - 用户说"这周六想去玩"、"明天周边哪里不下雨"等自然语言
  - Agent自动解析意图（时间+地点+需求）
- 自动定位：
  - 通过浏览器Geolocation API获取当前城市
  - 如果没权限或失败，询问用户或默认上海
- 智能日期解析：
  - 支持"这周六"、"下周日"、"明天"、"后天"、具体日期（2026-04-12）
  - 自动计算目标日期

**2. 动态计算周边城市**
- 基于出发城市，自动计算周边可达城市：
  - 数据来源：内置城市距离数据库（高铁/自驾时长）
  - 范围：高铁2小时内或自驾3小时内
  - 动态生成城市列表（如上海→杭州、苏州、无锡、嘉兴、宁波、南京、扬州、绍兴等）
- 支持用户手动添加/删除城市

**3. 详细天气数据**
- 天气API：使用免费额度充足的API（和风天气/OpenWeather）
- 展示维度：
  - 天气状况（晴/多云/阴/雨）
  - 温度范围（最高/最低）
  - 降雨概率
  - 湿度
  - 风力
  - 空气质量（如有）

**4. 可视化展示（混合式）**
- **地图热力图**：
  - 可交互的地图（高德/百度/Leaflet）
  - 标注周边城市位置
  - 用颜色深浅表示天气好坏（绿色=晴好，黄色=一般，红色=雨天）
  - 点击城市显示详情卡片
- **卡片网格**：
  - 每个城市一张卡片
  - 卡片内容：城市名、天气图标、温度、降雨概率
  - 按天气质量自动排序（晴天优先）
  - Top 3城市高亮显示

**5. 交互流程（混合式）**
- **主界面**：单页面，输入→地图+卡片结果
- **对话模式**：
  - 点击"询问Agent"按钮展开对话
  - 可追问："杭州有什么好玩的？"、"自驾去苏州要多久？"
  - Agent基于已有数据回答

**技术栈建议**：
- 前端：Next.js 14 (App Router) + React + TypeScript + Tailwind CSS
- 地图：Leaflet（开源免费）或高德地图JS API
- 天气API：和风天气（免费版：1000次/天）
- 交通API：
  - 公交/地铁/火车：**高德地图公交路径规划API**（免费额度：5000次/天）
  - 自驾：**高德地图驾车路径规划API**（免费额度：5000次/天）
  - 备选：百度地图API（类似额度）
- AI Agent：**多模型支持** + **自定义配置**
  - **模型适配层**：
    - 支持 OpenAI (GPT-4/GPT-3.5)
    - 支持 智谱AI (GLM-4/GLM-3)
    - 支持 阿里云 (通义千问)
    - 支持 Anthropic (Claude)
    - 统一接口封装，切换模型只需改配置
  - **自定义配置界面**：
    - 设置页面：输入API Key、选择模型、调整参数
    - 参数可调：Temperature（创造性）、Max Tokens（回答长度）
    - 本地存储：配置保存在浏览器LocalStorage（安全，不上传）
    - 环境变量：支持部署时预设默认配置
  - **Agent核心**：
    - Tool-use模式（天气、城市、交通等Tools）
    - Streaming流式响应
    - 多轮对话+上下文记忆
    - 主动引导用户澄清需求
- 部署：Vercel（免费+高性能）

**用户价值**：
- 3秒定位，5秒出结果
- 一眼看清周边哪里天气最好
- 点击即可查看详情
- 像和Agent聊天一样自然

### 阶段二：中度版 - 智能推荐
**目标**：在天气基础上，整合交通和内容信息，给出具体推荐

**新增功能**：
- **交通信息**：
  - 公交/地铁/火车：**高德地图公交路径规划API**
    - 输入：出发地+目的地+日期/时间
    - 输出：综合交通方案（地铁+公交+火车组合）
    - 展示：时长、价格、换乘次数、具体车次
  - 自驾：**高德地图驾车路径规划API**
    - 输入：出发地+目的地
    - 输出：驾车路线、距离、预计时间、过路费
  - 在详情卡片中展示最佳交通方案
  - 支持用户问Agent："去杭州高铁多久？"、"自驾去苏州要多久？"
- **小红书内容集成（MCP方式）**：
  - 实现方案：**自建小红书MCP Server**
    - 技术：Playwright + FastAPI + MCP SDK
    - 功能：搜索小红书笔记、提取标题/摘要/标签/点赞数
    - Tools：
      - `xiaohongshu_search_notes(city, keyword, limit)` - 搜索城市相关笔记
      - `xiaohongshu_get_note_detail(note_id)` - 获取笔记详情
      - `xiaohongshu_get_popular_tags(city)` - 获取城市热门标签
  - 展示方式：
    - 每个城市卡片显示2-3条热门笔记预览
    - 显示标题、关键图片、标签、热度
    - 点击可查看详情或跳转到小红书
- **智能推荐**：
  - 综合考虑：天气质量（40%）+ 交通便利度（30%）+ 内容热度（30%）
  - 给出Top 3推荐城市，附带推荐理由
  - 可展开对话，询问Agent为什么推荐这个城市

**技术集成**：
- 天气API：和风天气（免费额度）
- 地图：Leaflet或高德地图（可交互）
- 交通数据：自建数据库
- 小红书：自建MCP Server（Playwright抓取）

**技术集成**：
- 天气API
- 地图/交通API
- 小红书API/爬虫

**用户价值**：不仅知道去哪里，还知道怎么去、玩什么

### 阶段三：深度版 - 智能决策Agent
**目标**：理解用户偏好，提供完整的个性化方案

**新增功能**：
- **用户交互**：**AI Agent模式**
  - Agent架构：**Tool-use模式**
    - 用户输入自然语言（"这周六想去玩，哪里不下雨？"）
    - Agent理解意图，决定调用哪些Tools
    - Tools：
      - `get_location()` - 获取当前城市
      - `parse_date(query)` - 解析日期（"这周六"→2026-04-12）
      - `get_nearby_cities(city, maxDistance)` - 获取周边城市
      - `get_weather(city, date)` - 查询天气
      - `get_transit_route(from, to, date)` - 公交/火车方案
      - `get_driving_route(from, to)` - 自驾方案
      - `recommend_destination(preferences)` - 综合推荐
    - Agent综合所有Tool结果，生成自然语言回答
  - 对话流：
    - 支持多轮对话（"杭州怎么样？"、"高铁多久？"、"有什么好玩的？"）
    - 上下文记忆（记住用户在问哪个城市）
    - 主动引导（"你想去户外还是室内？"、"几个人？"）
  - 可视化增强：
    - Agent回答时，同步在地图上高亮相关城市
    - 点击城市，Agent自动讲解该城市的天气和交通
  - 如果没明确位置，自动定位当前城市
  - 如果没明确日期，询问或默认"这周末"
  - Agent理解用户偏好，主动询问细节（"你喜欢户外还是室内？"、"几个人去？"）
- AI分析：Agent分析天气、交通、小红书内容，匹配用户偏好
- 完整方案：输出一日游完整计划
  - 时间表（几点出发、几点到达、每个点玩多久）
  - 路线规划（地图可视化）
  - 打卡点清单（基于小红书热度）
  - 备选方案（如果天气变化）

**技术集成**：
- 天气API
- 地图/交通API
- 小红书API
- LLM（用于偏好理解和方案生成）

**用户价值**：一站式解决"去哪里、怎么去、玩什么"的完整决策

## 4. 目标用户

- 居住在一线/新一线城市的年轻人
- 周末有短途游需求
- 对天气敏感（不想下雨天出去玩）
- 习惯从小红书获取攻略

## 5. 核心竞争优势

1. **天气优先**：区别于传统旅游App，以天气为首要筛选条件
2. **周边聚焦**：专注"周边城市"场景，而非全量旅游目的地
3. **小红书整合**：对接小红书的内容生态，提供真实的UGC推荐
4. **决策导向**：不只是信息展示，而是帮用户做决策

## 6. 潜在挑战

### 技术层面
- 小红书API：官方API限制较多，可能需要爬虫方案
- 天气API：需要支持中国城市的准确预报
- 地图API：需要支持多种交通方式的路线规划

### 产品层面
- 冷启动：需要积累足够的城市覆盖和攻略内容
- 准确性：天气预测本身有不确定性
- 用户体验：如何平衡信息丰富度和决策效率

## 7. 下一步行动

1. **验证需求**：找5-10个朋友确认这个痛点是否普遍存在
2. **MVP范围**：确定轻度版的具体功能边界
3. **技术调研**：调研可用的天气API、地图API、小红书数据获取方案
4. **原型设计**：画出轻度版的产品原型和交互流程

#### 4. 动效设计

**过渡动画**：
- 页面加载：内容依次淡入（ stagger 100ms ）
- 消息出现：从底部滑入 + 淡入（ 300ms ease-out ）
- 卡片展开：高度动画 + 内容淡入（ 400ms spring ）
- 地图标记：缩放弹跳效果（ 200ms bounce ）

**微交互**：
- 按钮悬停：缩放 1.02 + 阴影加深
- 输入框聚焦：边框发光动画
- 加载状态：骨架屏 + 脉冲动画
- 成功状态：勾选标记 + 轻微弹跳
